# מסמך אפיון טכני - מערכת Vibe
**גרסה:** 1.0
**תאריך:** 5 בפברואר 2026
**סטטוס:** מוכן לפיתוח/העברה

---

## 1. סקירה כללית (Overview)
**Vibe** היא מערכת נגישות והתנהגות המיועדת לבני נוער (גילאי 13+), הפועלת כשכבת תצוגה (Overlay) על גבי ממשק וואטסאפ (Web/Mobile). המערכת מספקת משוב בזמן אמת על אופי השיחה, מתריעה בפני שפה פוגענית, ומחזקת התנהגות חיובית באמצעות משחוק (Gamification).

### מטרות המערכת
1.  הגברת המודעות לטון הדיבור בצ'אט.
2.  מניעת בריונות רשת באמצעות "פסי האטה" (Speed Bumps).
3.  יצירת חוויה חיובית באמצעות חיזוקים ויזואליים (Sparks).

---

## 2. הנחיות עיצוב (Design System)

### קונספט ויזואלי
*   **סגנון:** Cyber-soft / Futuristic Pop.
*   **אווירה:** טכנולוגית אך רכה, ידידותית ולא מאיימת.

### פלטת צבעים (Color Palette)
*   **Mint Green (חיזוק חיובי/רגיל):** `#34D399` (Tailwind: `emerald-400`)
*   **Soft Purple (רקעים/הדגשות):** `#A78BFA` (Tailwind: `violet-400`)
*   **Electric Blue (פעולות/מידע):** `#60A5FA` (Tailwind: `blue-400`)
*   **Glowing Orange (אזהרה/Vibe נמוך):** `#FB923C` (Tailwind: `orange-400`)
*   **רקע כללי:** כהה/חצי שקוף (Glassmorphism) כדי להשתלב מעל וואטסאפ.

### טיפוגרפיה ושפה
*   **שפות:** עברית (He) - ברירת מחדל, אנגלית (En).
*   **כיווניות:** תמיכה מלאה ב-RTL (מימין לשמאל) עבור עברית ו-LTR עבור אנגלית. הממשק כולו חייב להתהפך בהתאם לשפה שנבחרה.

## 3. רכיבי מערכת (Core Components)

### 3.1. Vibe Meter (מד האווירה)
*   **תיאור:** בועה צפה (Floating Action Button) הנמצאת תמיד בפינת המסך.
*   **פונקציונליות:**
    *   מציגה ציון בזמן אמת (0-100) של "אווירת הצ'אט".
    *   משנה צבע בהתאם לציון (ירוק = טוב, כתום/אדום = רעיל).
    *   לחיצה עליה פותחת את התפריט הראשי/דשבורד.

### 3.2. Speed Bump (פס האטה - מנגנון אזהרה)
*   **טריגר:** מופעל כאשר המשתמש מקליד מילים המזוהות כרעילות או אגרסיביות.
*   **התנהגות:**
    *   הופעת Overlay החוסם רגעית את כפתור השליחה או מופיע מעל תיבת הטקסט.
    *   הודעה: "היי, בטוח שזה מה שרצית להגיד?" (או ניסוח דומה).
    *   אפשרות לניסוח מחדש (Edit) או שליחה בכל זאת (Send Anyway - לאחר השהייה).

### 3.3. Vibe Spark (חיזוק חיובי)
*   **טריגר:** שימוש במילים חיוביות, פרגון, או שמירה על Vibe Meter גבוה לאורך זמן.
*   **התנהגות:**
    *   אנימציית חלקיקים (Confetti/Sparks) על המסך.
    *   קבלת ניקוד (XP) שמוצג למשתמש.

### 3.4. דשבורד ותקציר יומי (Dashboard)
*   מסך סיכום המציג:
    *   גרף של ה-Vibe היומי.
    *   כמות ה-Sparks שנצברו.
    *   המלצות לשיפור ("נסה להיות יותר סבלני בבוקר").
    *   אוסף הסטיקרים שנפתחו.

### 3.5. בורר שפה (Language Toggle)
*   כפתור בולט בממשק המאפשר מעבר מיידי בין עברית לאנגלית.
*   **דרישה טכנית:** שינוי גלובלי של ה-Context, היפוך כיוון (Flex-direction, Text-align) ושינוי כל הטקסטים במערכת.

## 4. תרחישי משתמש (User Flows)

### Flow 1: זיהוי רעילות (Toxic Detection)
1.  המשתמש מקליד הודעה בוואטסאפ: "אתה ממש טיפש".
2.  המערכת מזהה מילת מפתח שלילית.
3.  **Speed Bump מופעל:** הבועה משתנה לכתום, הודעת אזהרה קופצת.
4.  המשתמש בוחר "ערוך הודעה".
5.  המשתמש משנה ל: "לא הבנת אותי נכון".
6.  המערכת מזהה תיקון -> **Vibe Spark מופעל** (חיזוק על תיקון).

### Flow 2: החלפת שפה (Language Switch)
1.  המשתמש לוחץ על כפתור הדגל/שפה (EN/HE).
2.  **State Update:** משתנה השפה מתעדכן ב-Global State.
3.  **UI Update:**
    *   כל הטקסטים מתורגמים מיידית.
    *   ה-Layout מתהפך (התפריט עובר מצד ימין לשמאל או להיפך).
    *   ההודעות המדומות (Mock Data) מתחלפות לשפה המתאימה.

### Flow 3: צפייה בסיכום יומי
1.  המשתמש לוחץ על ה-Vibe Meter.
2.  נפתח תפריט צף.
3.  המשתמש בוחר "סיכום יומי".
4.  מוצג פופ-אפ עם גרף ומדדים.

## 5. דרישות טכניות (Technical Stack)
*   **Frontend:** React 18+.
*   **Styling:** Tailwind CSS v4 (שימוש ב-Utility classes לניהול RTL בצורה נקייה, למשל `s-0` במקום `left-0`).
*   **Icons:** Lucide-React.
*   **Animation:** Motion (Framer Motion).
*   **Charts:** Recharts (עבור הסיכום היומי).
*   **State Management:** React Context API (לניהול שפה ונושא).

## 6. נכסים (Assets)
*   יש לוודא שימוש ב-SVG עבור אייקונים ליכולת שינוי צבע דינמית.
*   תמונות פרופיל ורקעים ייטענו ממקורות חינמיים (Unsplash) או נכסים מקומיים.

**הערות למפתח:**
*   שים לב שהמערכת היא Overlay - הרקע של ה-Container הראשי צריך להיות שקוף או חצי שקוף כדי לדמות את החוויה מעל וואטסאפ.
*   חשוב מאוד להקפיד על נגישות (Contrast) בבחירת הגוונים של הטקסט על הרקעים הכהים.
